# Data Source and Reliability Inventory

## 1. Purpose

This document provides a structured inventory of the key inputs, data sources, and assumptions used in the comparative model for road-only and multimodal (road–cabotage–road) freight transport. By transparently mapping what data is observed versus assumed, this inventory supports the methodological robustness of the final academic report (Trabalho de Formatura - TF). It ensures that uncertainties, approximations, and data limitations are explicitly acknowledged and appropriately handled in the final text.

## 2. Reliability classification

For the purpose of this study, input sources and their reliability are classified as follows:

**Source Types:**
- **Official/observed data:** Data collected from official regulatory bodies, market indices, or empirical reporting frameworks (e.g., ANTT, EU MRV, Ship & Bunker).
- **API-calculated data:** Output generated by deterministic routing engines or third-party APIs (e.g., OpenRouteService).
- **Literature/technical factor:** Standardized coefficients from academic literature, industry guidelines, or engineering standards (e.g., IPCC/IMO emission factors, EMEP/EEA guidebook).
- **Calibrated parameter:** Variables adjusted to fit the specific operational context of the Brazilian transport matrix, based on technical reports or sample datasets.
- **Modeling assumption:** Deterministic values chosen to construct valid scenarios, such as baseline payloads or operational simplifications.
- **Unknown:** Sources that cannot be definitively traced to a public or official origin within the repository.

**Reliability Levels:**
- **High:** Data derived directly from authoritative, well-documented, or recently updated sources with low ambiguity.
- **Medium:** Credible technical factors, API outputs, or literature values that may carry contextual uncertainties (e.g., applying European averages to the Brazilian context) or modeling simplifications.
- **Low:** Proxies, outdated values, or broad assumptions that introduce significant uncertainty to the model's accuracy.
- **Unknown:** The traceability of the data point is insufficient to assess its reliability.

## 4. Key methodology debts

The following items represent areas where the current repository state contains legacy implementations, ambiguities, or gaps that should be addressed or documented prior to finalizing the thesis:

- **Cost Boundaries:** The model currently derives costs predominantly from updated fuel price inputs, which may underestimate true comprehensive transport tariffs (e.g., tolls, port fees, labor, depreciation). This gap must be explicitly documented.
- **Legacy Port Submodels:** Legacy models (e.g., `calcs/hotel.py` and `modules/fuel/cabotage_fuel_service.py`) persist in the repository. They utilize older abstractions that conflict with the active MRV-based and EMEP/EEA hoteling models. These must be marked as legacy/deprecated to prevent mixed-method runs.
- **Well-to-Wake (WTW) Scope:** The current emission factors (`modules/fuel/emissions.py`) are strictly Tank-to-Wheel (TTW). Failing to model the upstream extraction and refining phases prevents a complete lifecycle comparison between diesel and marine fuels.
- **Dynamic Routing Variations:** Reliance on static distance matrices (`sea_matrix.json`) or simplified haversine fallbacks ignores the variability of real-world ship routing and weather-induced detours.

## 5. Recommended thesis wording

To maintain academic rigor and transparency, the following wording is suggested for the methodology and limitations sections of the final text:

> *"The quantitative comparison relies on a combination of empirical datasets, standardized technical factors, and deterministic modeling assumptions. Where possible, empirical data was prioritized—such as utilizing the EU MRV database as a proxy for vessel efficiency and referencing Ship & Bunker for current marine fuel prices. However, projecting European operational profiles and theoretical equipment efficiencies onto the Brazilian transport network introduces necessary limitations. Road routing parameters rely on API-calculated shortest paths and ANTT-derived baseline efficiencies, which do not continuously capture dynamic operational variables like traffic congestion or road grading. Furthermore, the emission inventory is constrained to Tank-to-Wheel (TTW) boundaries. To address these uncertainties, the methodology treats assumed parameters as static baselines and recommends interpreting absolute values with caution, utilizing sensitivity analyses for highly variable inputs such as fuel prices and empty backhaul configurations."*
