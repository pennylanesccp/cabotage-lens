# TF Literature Citation Map

## 1. Purpose and use

This document maps audited literature sources to safe uses in the final Trabalho de Formatura (TF) report and later technical article. It is a citation-boundary handoff, not a methodology update.

Use this map to decide which sources may support:

- background;
- methodology;
- validation;
- sensitivity;
- limitations;
- future work;
- sources that should not be used directly in the model.

This document does not introduce new model assumptions, coefficients, emission factors, fuel factors, distances, costs, routes, ports, validation verdicts, sensitivity outputs, or results. Literature values listed here are citation targets or boundary warnings only unless a separate methodology issue explicitly aligns the boundary, unit, and implementation path.

The current project boundary remains operational TTW CO2e and model cost estimates. WTW, LCA, CO2-only, commercial freight-rate, supernetwork, schedule, frequency, HVO, and port-specific hotelling references must stay separated unless the TF text explicitly labels them as comparison, limitation, sensitivity, or future-work material.

## 4. Do not cite for this

| Do not cite this source or evidence for... | Because... | Safer alternative |
| --- | --- | --- |
| WTW/LCA papers such as `competitiveness2024`, `decarb2024`, or `maritimelca2024` to claim TTW model validation. | Their values include upstream or lifecycle components, or remain pending audit. | Cite them as boundary contrast/future work, and cite project validation artifacts for current TTW CO2e outputs. |
| `competitiveness2024` to claim CabotageLens cost output is a market freight rate. | That study includes commercial/network cost components not represented in the current model boundary. | Say CabotageLens reports model cost estimates/proxies and cite `docs/tf_system_boundary.md`. |
| `icct2022` national average road/cabotage rates to validate route-level CabotageLens costs. | National averages and freight-rate statements are not corridor-specific model outputs. | Use as background/economic context only. |
| Alternate-port sensitivity rows to claim exact selected-port validation. | Pecem is not Porto de Fortaleza; Suape is not Porto do Recife. | Label them as alternate-port sensitivity rows using `tf_sensitivity_results_batch_001b.md`. |
| Haversine or fallback-only maritime distances to claim a validated cabotage corridor. | Fallback distance is a screening estimate and may bias route cost/emissions. | Use exact selected-port evidence or explicitly named sensitivity assumptions. |
| `shortsea2019` or other short-sea comparison papers to prove cabotage is always better. | The audited conclusion is conditional on utilization, vessel size, corridor, and assumptions. | Use the paper to support corridor-specific caution. |
| Port hotelling papers (`berth2009`, `shipops2022`, `berthairquality2010`) as complete route-level modal-shift validation. | They address at-berth/port-side methods, not full road-vs-cabotage route outcomes. | Use them for port/hotelling methodology or limitations only. |
| `decarb2024` HVO results to reduce current baseline emissions. | HVO is an alternative-fuel WTW scenario, not the current TTW baseline. | Use as future work or a separately bounded scenario proposal. |
| `isoemission2019` to claim CabotageLens already has iso-emission mapping. | The current app does not implement a full iso-emission map and the audit is pending. | Use as future visualization/mapping work only. |

## 5. Handoff to issue #19

Use this citation map in the final TF report as follows:

- Introduction/background: use `icct2022` for Brazilian cabotage, BR do Mar/policy context, modal share, and national emissions/cost motivation. Use `competitiveness2024` only as additional Brazilian cabotage competitiveness context.
- Literature review: organize around modal shift (`modalshiftreview2020`), SSS versus road conditionality (`shortsea2019`, `isoemission2019` after audit), Brazilian supernetwork work (`competitiveness2024`), port/hotelling sources (`berth2009`, `shipops2022`, `berthairquality2010`), and WTW/LCA/fuel pathways (`decarb2024`, `maritimelca2024` after audit).
- Methodology: cite project boundary docs first (`docs/tf_system_boundary.md`, `docs/port_ops_model.md`, and `docs/tf_literature_audit_implementation_plan.md`). Use literature only to justify why the boundary is explicit and why broader components are excluded or marked future work.
- Results: rely primarily on project artifacts (`docs/validation/tf_final_result_synthesis.md`, `docs/validation/tf_sensitivity_results_batch_001b.md`, `docs/validation/tf_validation_batch_001b_methodology_decisions.md`, and `docs/validation/tf_validation_batch_001b_results.md`). Do not replace project results with literature values.
- Discussion: compare the sensitivity and limitation pattern against literature cautiously. Emphasize corridor-specific outcomes, maritime-distance provenance, port-selection sensitivity, cost-boundary limits, and operational TTW CO2e boundaries.
- Limitations/future work: use WTW/LCA, HVO, commercial freight rates, schedules/frequencies, supernetwork modeling, Brazilian port productivity data, and full iso-emission mapping as future-work categories unless a later issue explicitly expands the model boundary.
