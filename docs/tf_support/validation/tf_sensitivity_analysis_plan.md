# Sensitivity Analysis Plan

## 1. Purpose

Sensitivity analysis tests whether the thesis conclusions survive plausible variation in uncertain parameters. For CabotageLens, this is especially important because road-only and road-cabotage-road alternatives depend on fuel prices, fuel consumption, maritime intensity, distance approximations, port operations, load assumptions, and cost-boundary choices.

This document is a planning note only. It does not run sensitivity analysis, report results, or define final thesis coefficients. Any numeric range below is an initial planning range that must be reviewed against sources, validation results, and the final thesis case-study boundary before use.

This plan should be read together with the existing TF planning documents: `docs/tf_support/methodology/tf_assumptions_and_approximations.md`, `docs/tf_support/validation/tf_validation_plan.md`, `docs/tf_support/methodology/tf_data_reliability_inventory.md`, and `docs/tf_support/validation/tf_result_classification_rules.md`, as applicable.

## 3. Scenario matrix

The final thesis should avoid an excessive scenario grid. A compact matrix is easier to interpret and easier to defend. The scenarios below are proposed for planning only; final parameter movements must be source-reviewed.

| Scenario | Parameters moved | Rationale | Recommended interpretation |
| --- | --- | --- | --- |
| Base scenario | Current model inputs and current documented boundary. Record all input values, data sources, route source flags, cargo mass/TEU, and whether hoteling/port ops are included. | Main reproducible thesis case. | Use as the central estimate, not as the only evidence. |
| Cabotage-favorable scenario | Higher diesel price, lower bunker price, less efficient truck fuel consumption, more efficient maritime intensity, no empty backhaul penalty assigned to multimodal access unless justified, and validated maritime distances. | Tests whether cabotage advantage strengthens under plausible favorable assumptions. | Do not present as expected reality; present as an upper-bound or favorable sensitivity. |
| Road-favorable scenario | Lower diesel price, higher bunker price, more efficient truck fuel consumption, less efficient maritime intensity, added hoteling/port-ops burden where methodologically valid, and longer maritime distance for uncertain pairs. | Tests whether cabotage conclusions survive assumptions favorable to road-only transport. | If cabotage still wins here, the result is more robust. |
| Conservative emissions scenario | Use higher road and maritime emission factors only when source-supported, include empty backhaul if system boundary requires it, include hoteling/port ops when not double counted, and retain tank-to-wheel boundary unless WTW factors are explicitly added. | Tests climate conclusions under conservative direct-emissions assumptions. | Useful for emissions ranking; do not mix TTW and WTW factors without labeling. |
| High fuel-price volatility scenario | Move diesel and bunker prices in opposite directions and together in separate subcases; keep physical fuel quantities fixed. | Tests whether economic conclusions are mainly fuel-price artifacts. | If cost ranking changes but emissions ranking does not, report cost sensitivity separately from environmental conclusion. |
| Wide cost-boundary scenario | Add sourced non-fuel costs such as tolls, port tariffs, inventory/time, service-frequency penalty, or handling cost if available. | Tests whether simplified energy-cost results are stable when the economic boundary expands. | If unsupported by sources, keep as methodology debt rather than final result. |
| Data-quality conservative scenario | Use only OD/port pairs with validated distances and observed route intensity; exclude or flag fallback-heavy routes. | Tests whether conclusions depend on lower-confidence routes. | Use to distinguish empirical-core conclusions from exploratory route outputs. |

## 4. Interpretation rules

- Robust: The same qualitative conclusion remains valid across plausible low/base/high parameter ranges and across road-favorable and cabotage-favorable scenarios. Example: cabotage remains lower-emission for an OD pair even under road-favorable assumptions.
- Sensitive: The conclusion changes under plausible parameter ranges. Example: cost advantage switches between modes when diesel or bunker price varies within reviewed ranges.
- Inconclusive: The difference between alternatives is small compared with parameter uncertainty, validation tolerance, or data quality limits. Report the route as not decisively favoring either mode.
- Invalid/out-of-scope: Route logic, data coverage, geocoding, distance source, or operational feasibility is insufficient for a defensible conclusion. Do not use the case for headline results.
- Boundary-dependent: The conclusion is valid only under a stated boundary, such as fuel-only cost, tank-to-wheel emissions, no empty backhaul, or nearest-port geometry. Label the boundary in every table and figure.
- Fallback-dependent: The conclusion relies on class fallback, distance fallback, or uncertain service plausibility. Downgrade confidence and require validation or sensitivity before thesis claims.

## 5. Recommended outputs for final TF

- Sensitivity table: For each final OD pair, show base result, low/high changes for the high-priority parameters, and whether the mode ranking changes.
- Tornado chart: Use one chart per representative OD pair to show which parameters most affect emissions savings and cost savings. Keep cost and emissions charts separate.
- Scenario comparison table: Compare base, cabotage-favorable, road-favorable, conservative emissions, and fuel-price volatility scenarios.
- Robust/sensitive/inconclusive classification by OD pair: Assign each case a conclusion label and explain the reason in one sentence.
- Boundary note below every table: State whether the result is fuel-only cost, energy plus port operations, direct tank-to-wheel emissions, or another boundary.
- Fallback/quality columns: Report route distance source, maritime intensity source, MRV/ANTAQ match status where available, hoteling treatment, port-ops treatment, and whether nearest-port logic was accepted or overridden.
- Unit-normalized indicators: Include per-shipment totals and normalized indicators such as kg CO2e/tkm or kg CO2e/t where appropriate, with km and nm conversions documented.

## 6. Methodology debts

- Verify final base values and sources for diesel price, bunker price, emission factors, truck fuel consumption, maritime fuel intensity, cargo mass/TEU, load factor, and port operation assumptions.
- Define final low/high ranges from observed data, literature, validation results, or benchmark sources rather than relying on initial planning percentages.
- Decide whether empty backhaul belongs in the main boundary, a sensitivity-only boundary, or a limitation.
- Decide whether carbon price or emissions monetization is used at all. If used, cite the source and keep the physical emissions result separate from monetized cost.
- Decide whether cost analysis remains energy-cost-only or expands to tolls, port tariffs, inventory cost, handling charges, time, and service frequency.
- Validate maritime distance sources before applying distance-factor sensitivity to final conclusions.
- Validate truck fuel consumption ranges against Brazilian heavy-vehicle references or operator data.
- Confirm whether hoteling is excluded because route transport-work intensity already covers operational fuel, or included as a separate class/fallback decomposition.
- Review port cargo-handling placeholders before using port-ops sensitivities as final thesis evidence.
- Prepare a reproducible sensitivity run manifest including code version, input payload, data artifact versions, scenario parameters, and output timestamp.

## 7. Recommended thesis wording

The following wording can be adapted for the methodology or sensitivity-analysis chapter.

> Sensitivity analysis was used to test whether the relative performance of road-only and road-cabotage-road alternatives depends on uncertain parameters. The objective was not to identify a single exact value, but to evaluate whether the qualitative conclusion remains stable under plausible changes in fuel prices, fuel consumption, maritime intensity, distance assumptions, load assumptions, hoteling, empty backhaul, and cost boundary.

> Parameters were varied only within ranges that were either source-supported or explicitly identified as initial planning ranges requiring review. When a result changed under plausible assumptions, the route was classified as sensitive rather than robust.

> Cost and emissions were interpreted separately. A route may be robustly lower-emission while remaining cost-sensitive if fuel prices or non-fuel logistics costs are uncertain.

> Results based on fallback maritime intensity, fallback distances, unvalidated nearest-port choices, or unsupported cost-boundary assumptions were not treated as strong evidence. They were classified as exploratory, sensitive, or methodology debt depending on the route.
